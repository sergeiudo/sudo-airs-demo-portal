Flows & logic

Three flows — same prompt, same model

FlowPathConfigCatches
1 · No gatewayApp → Vertex directly (bypasses Portkey)nonenothing
2 · AI GatewayApp → Portkey → Vertexpc-sudo-d-a72af9PII (redact) · banned words · code
3 · Gateway + AIRSApp → Portkey → AIRS → Vertexpc-sudo-t-5a505binjection · jailbreak · harmful · DLP · URLs
One key, per-request config. The portal sends one key + an x-portkey-config per request — so it needs Allow Config Override = ON. Off = every lane collapses to one config.
Two jobs. Native guardrails enforce business / data policy; AIRS catches the AI-native threats regex/PII can't see. No-gateway is the "before".

Setup — what to configure

The four building blocks

① Integrations · @sudo-vertexai + @sudo-bedrock

  • Vertex · SA-file · project sergei-playground-338006 · region global (gemini-3.x is global-only); provision gemini-3.1-flash-lite + gemini-3.5-flash
  • Bedrock · Assumed Role · region us-west-2; provision Claude·DeepSeek·Qwen·Kimi·Nemotron — same configs route to whichever provider the picked model belongs to

② Key · sudo-demo-portal-full-key

  • Value in .env as PORTKEY_API_KEY
  • Allow Config Override = ON — the linchpin
  • Lets the portal pick native / AIRS per request

③ Native guardrail · pg-sudo-p-c7e4ad

  • No PII redact EMAIL·PHONE·ADDR·NAME·IP·CC·SSN
  • Contains Code Python · Inverse ON (block on code)
  • Contains none-op: סרגיי·Chekpoint·Kokomoko

④ AIRS guardrail · pg-sudo-a-c3bfdd

  • PANW Prisma AIRS · profile sudo-airs-api-profile-new
  • App user sudo-app · block on action=block
  • Tune detections in Strata Cloud Manager

→ Each guardrail is wrapped in a config (input + output) with no override_params, so the request's model drives all lanes. { retry:3, cache:simple, provider:@sudo-vertexai, in/out_guardrails:[…] }

.env, deploy & gotchas

# sudo-demo-portal-full-key — override ON
PORTKEY_API_KEY=<full-key secret>
PORTKEY_CONFIG_AIRS=pc-sudo-t-5a505b
PORTKEY_CONFIG_DEFAULTS=pc-sudo-d-a72af9
PORTKEY_CONFIG_NO_GUARDRAIL=pc-sudo-d-a72af9
PORTKEY_CONFIG_FALLBACK=
PORTKEY_VERTEX_SLUG=@sudo-vertexai
PORTKEY_BEDROCK_SLUG=@sudo-bedrock
# EC2 deploy (origin: sudo-airs-demo-portal)
cd /opt/sudo-airs-demo && git pull origin main \
  && npm run build && pm2 restart airs-server
# .env is gitignored — hand-edit on EC2

Gotchas

  • Override ON on the portal key — or lanes collapse.
  • gemini-3.x = global; direct lane uses the Vertex OpenAI endpoint + google/ prefix.
  • No override_params → model picker drives all lanes.
  • Model must be provisioned in Portkey AND Vertex (else model_not_allowed / 404).
  • Guardrails: Code Inverse=ON · Contains op=none · PII = redact.
  • Redaction shows as ALLOWED · REDACTED (transformed:true).
