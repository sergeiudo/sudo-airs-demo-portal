Pillar 1
API Intercept
Every prompt and response is scanned by Prisma AIRS before the LLM sees it and before the user does. Threats are caught in real time — prompt injection, jailbreaks, sensitive data leaks.
Why it matters
Customers don't control what users send to their AI. AIRS is the last line of defense before a prompt becomes an attack.
How to Demo
1
Navigate to "API Intercept" from the home screen
Show them the 3-column layout — attack library, chat, and telemetry panel.
2
Toggle protection OFF — send a jailbreak from the attack library
The model responds. Point out: no scan, full exposure. This is the "before" state.
3
Toggle protection ON — send the same attack
AIRS blocks it. Show the telemetry: threat category, scan latency, profile used.
4
Click "View in SCM Console" in the sidebar
This deep-links to Strata Cloud Manager — the same event, full forensics, policy controls.
Pro tip: Use the "Prompt Injection" category first — it's the most visually dramatic. Then switch to "Data Exfiltration" to show breadth.
2
Pillar 2
Model Scanning
AI models are files — and files can be weaponized. Before any model is deployed, Prisma AIRS scans it for embedded malware, backdoors, and supply chain compromises.
Why it matters
Models downloaded from HuggingFace or internal registries are blind trust. One poisoned model can own your entire AI stack.
How to Demo
1
Navigate to "Model Scanning" from the home screen
Show the model registry on the left — these are models as they'd appear in a real environment.
2
Select a model and click "Scan Model"
Watch the radar animation — this represents deep inspection: weights, serialized objects, embedded payloads.
3
Select the pre-compromised model and scan it
The vulnerability report appears — show the findings: CVE references, severity, remediation guidance.
4
Compare clean vs. compromised scan results side by side
Same model family, different source. This is a supply chain attack in the wild.
Pro tip: Lead with the question "Do you know what's in the models your teams are downloading?" — it reframes the conversation from tool to risk.
3
Pillar 3
Red Teaming
Automated adversarial campaigns probe your AI for vulnerabilities before attackers do. Hundreds of attack variants, real-time robustness scoring, exportable reports.
Why it matters
Manual red teaming is slow and expensive. Prisma AIRS runs thousands of attacks in minutes and gives you a quantified robustness score.
How to Demo
1
Navigate to "Red Teaming" — show the Campaign Builder
Point out: target selection, attack categories, intensity settings. This is how security teams configure a campaign.
2
Launch a campaign against the configured target
Watch the live attack log feed — each entry is a real adversarial probe with pass/fail result.
3
Watch the Robustness Gauge update in real time
This is a quantified security score — not a vague "your AI is secure." Executives love this.
4
Show the campaign report when complete
Attack breakdown, failed probes, remediation priorities — ready to share with security leadership.
Pro tip: If credentials are set up, run against a real target. If not, the live log feed still tells the story — focus on the robustness gauge narrative.
4
Pillar 4
IDE Protection
Claude Code hooks intercept AI-generated code before it lands on disk. Every file write triggers a Prisma AIRS scan — insecure code is flagged or blocked before the developer even sees it.
Why it matters
Developers trust AI coding assistants blindly. AIRS makes every AI-generated line of code pass a security gate — automatically, at dev time.
How to Demo
1
Navigate to "IDE Protection" — open the Claude Code Hooks guide
This is the integration guide your customers' devs would follow. Walk through it top to bottom.
2
Show the hook configuration — PreToolUse on Write/Edit events
One JSON config, drops into any Claude Code project. Zero code changes to their app. This is the "5-minute install" story.
3
Walk through the scan flow: hook fires → AIRS scans → result returned to Claude
If AIRS flags the code, Claude is told to stop. The bad file never gets written. Show the shell script that does this.
4
Show an example block event — insecure code caught pre-write
Ask: "At what point in your SDLC do you currently catch AI-generated vulnerabilities?" Then let the answer sell it.
Pro tip: This pillar resonates most with DevSecOps personas. Lead with "shift-left AI security" framing — they already speak that language.
